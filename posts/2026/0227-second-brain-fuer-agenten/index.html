Second Brain für Agenten

2026-02-27 Drinnen

Nach drei Wochen im täglichen Einsatz möchte ich mein Agent-Skill Plugin agent-todos nicht mehr missen. Es führt einfach zu so viel besseren Ergebnissen, wenn man Claude Code oder auch Codex wohlüberlegte, strukturierte To-Dos zum Abarbeiten gibt, als spontane Prompts zu verwenden. Und zumindest Claude hat auch damit angefangen selbsständig Folge-To-Dos anzulegen.

Nur die Tatsache, dass ich die To-Dos direkt in den Projekten abgespeichert habe, hat sich als unpraktikabel erwiesen. Spätestens als ich anfing, sie auch für Projekte bei der Arbeit zu verwenden, kam dann auch noch die Frage der Akzeptanz von anderen Teammitgliedern ins Spiel.

So habe ich die Skills so anpassen lassen, dass sie die To-Dos in ein konfigurierbares Verzeichnis schreiben und von dort lesen. Dieses Verzeichnis kann außerhalb des Projekts und somit auch in einem Obsidian-Vault liegen.

Handelt es sich bei dem konfigurierten Verzeichnis um ein Obsidian-Vault, kann zusätzlich noch konfiguriert werden, dass bei Veränderungen im Status von To-Dos ein Kanban-Board aktualisiert wird. Somit kann ich mir einen schnellen Überblick über alle To-Dos und deren Status verschaffen. Allerdings wird hierfür das Kanban-Plugin für Obsidian benötigt.

Jetzt wünsche ich mir noch, dass To-Dos automatisch umgesetzt werden, wenn ich sie auf dem Kanban-Board auf Doing verschiebe, so wie es das sehr coole Tool Clave tut. Mal schauen, ob das eine Bastelei für den kommenden Freitag wäre.
