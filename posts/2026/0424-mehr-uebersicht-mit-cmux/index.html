Mehr Übersicht mit cmux

2026-04-24 Drinnen

Was mich schon länger umtreibt, ist die Frage: Wie behalte ich die Übersicht, wenn ich mehrere Instanzen von Claude Code und/oder Codex parallel arbeiten lasse? Zufääligerweise haben wir darüber auch am Donnerstag in unserer Software Engineering Community of Practice (COP) bei der Arbeit geredet. Bisher hatte ich immer mehrere Tabs im Terminal oder in VSCode offen. Ab einer bestimmten Anzahl verlor ich immer den Überblick. Auch Claude Features wie /rename und /color halfen nur bedingt, da ich ja trotzdem zuerst ein Tab öffnen musste, um zu sehen, was sich dahinter verbirgt. Außerdem bekam ich nie mit, wenn eine der Instanzen auf User-Input wartete.

Nun habe ich heute die Terminal App cmux ausprobiert und ich bin begeistert. Sie basiert auf dem Terminal Emulator ghostty und verspricht Multitasking für Menschen und Coding-Agenten. Statt Tabs für einzelne Terminalfenster gibt es vertikale Reiter, die individuell benannt und farblich markiert werden können. Diese werden in cmux Workspaces genannt und können unterschiedliche horizontale und vertikale Bereiche enthalten. Jeder Bereich kann wiederum ein oder mehrere Terminals oder Inline-Webbrowser enthalten. So kann man zum Beispiel links mit Claude Code an einem Frontend Task arbeiten und rechts daneben direkt das gerenderte Ergebnis im Browser begutachten.

Dank Kitty Image Protocol können auch Bilder im Terminal angezeigt werden. Ein weiteres Feature sind Benachrichtigungen, die informieren, sobald Agenten auf Input warten. In Kombination mit CLI Tools wie Yazi oder Lazygit lassen sich so auch ganze Workflows im Terminal abbilden. Oder man baut sich Dashboards, die Logfiles und Monitoring-Tools nebeneinander anzeigen. Die Workspaces bleiben auch nach einem Neustart von cmux erhalten. Ich werde da in nächster Zeit wohl einiges ausprobieren.
