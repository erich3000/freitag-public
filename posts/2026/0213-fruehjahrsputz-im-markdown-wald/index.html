Frühjahrsputz im Markdown-Wald

2026-02-13 Drinnen

Schon im September letzten Jahres hatte ich meine Rezeptsammlung in Obsidian in eine Base-Datenbank überführt. Ich muss aber zugeben, dass ich seither noch nicht viel gemacht habe, um eine bessere Auffindbarkeit einzelner Rezepte zu ermöglichen. Zu mühsam erschien es mir, Tags per Hand zu vergeben und die sogenannten Properties in Obsidian zu setzen. So vegetierte die Sammlung genauso unbenutzt vor sich hin, wie sie es zuvor schon auf Trello getan hat.

Vor kurzem las ich dann auf medium.com, dass es ein Claude-Plugin mit Agent-Skills für Obsidian gibt, das unter anderem Skills für Base-Verwaltung und Markdown-Formatierung bereitstellt.

Das hat mich neugierig gemacht. Also habe ich Claude Code Zugang zu meiner Vault gegeben und die Obsidian-Skills installiert. Für ein besseres Handling habe ich die Vault dann das erste Mal in VS Code geöffnet. s Und dabei fiel es mir wie Schuppen von den Augen: Die ominösen Obsidian-Properties sind nicht weiter als YAML-Frontmatter-Felder, wie ich sie fast täglich in den Markdown-Dateien für meinen Hugo-Blog vor mir habe. Solche Frontmatter-Felder lassen sich natürlich viel einfacher auch in größeren Mengen hinzufügen oder anpassen, als dies in Obsidian selbst möglich ist.

In Kombination mit dem Skill /obsidian-markdown aus dem Plugin konnte ich so endlich Ordnung reinbringen, indem ich rezeptspezifische Felder wie zum Beispiel cuisine und yield einführte, wie man sie aus Recipe-Schema kennt, und mithilfe von Claude Code Tags generierte.

Diese neuen Felder habe ich dann mit der Hilfe des Skills /obsidian-bases als Filter zur Datenbank hinzugefügt. Außerdem hat mir Claude die Anzeige von Thumbnails in der Datenbank angepasst.
